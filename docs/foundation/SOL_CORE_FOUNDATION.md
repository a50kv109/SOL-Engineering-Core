# SOL Core Foundation

## 1. Назначение
SOL (Structural Operational Language) — фундаментальное онтологическое ядро E-OS. Предназначено для обеспечения математически строгой базы для описания инженерных сущностей, их ограничений и изменений.

## 2. Аксиомы (Immutable)
1. **SOURCE:** Инициация потенциала.
2. **CONSTRAINT:** Структура определяется границами.
3. **DISSIPATE:** Обновление требует рассеивания энтропии.
4. **COMPARE:** Валидация требует сопоставления состояний.

## 3. Kernel Primitives
- **SOURCE:** Генератор потенциала (энергии, данных, состояния).
- **CONSTRAINT:** Фильтр допустимых состояний.
- **COMPARE:** Оператор сопоставления состояний.
- **DISSIPATE:** Оператор утилизации/забывания состояний.

## 4. Design Principles (Kernel)
- **Zero-Domain Knowledge Injection:** Ядро не содержит специфических знаний о домене (подшипниках, сварке и т.д.).
- **No Compute inside SOL:** SOL только описывает условия, вычисления выполняет Runtime.
- **Separation of Concerns:** SOL (онтология) изолирован от ISA (типизация) и Runtime (исполнение).
