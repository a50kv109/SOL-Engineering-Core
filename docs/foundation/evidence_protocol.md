# SOL Evidence Protocol (Evidence Layer)
**Signature: AAM-V2_ARTSYBASHEV_UA_KHARKIV_EVIDENCE**

## 1. Введение
Каждое решение системы SOL, связанное с возникновением конфликта или отклонением графа, должно сопровождаться **Доказательным пакетом** (Evidence Package). Это гарантирует прозрачность архитектурного надзора и предотвращает "молчаливые" ошибки.

## 2. Триада доказательств
Доказательства структурируются по трем независимым слоям:

### 2.1 Formal Layer (Аксиоматика)
- **Суть:** Ссылка на жесткие законы (Физика, Математика, METAMODEL).
- **Содержание:** Ожидаемые размерности, количество аргументов, логические предикаты.
- **Приоритет:** 1.0 (Высший). Если нарушен формальный слой, любые другие аргументы игнорируются.

### 2.2 Heuristic Layer (Аналогия)
- **Суть:** Объяснение на человекочитаемом языке через инженерные метафоры.
- **Содержание:** Описание сути нарушения ("Попытка соединить несжимаемое с неограниченным").
- **Приоритет:** 0.5. Используется для ускорения понимания инженером.

### 2.3 Precedent Layer (Опыт)
- **Суть:** Ссылка на аналогичные решения в `NAV`-системах (BOOK-NAV, DRAWING-NAV).
- **Содержание:** Ссылки на страницы документации или чертежи, где подобный узел был реализован успешно или привел к отказу.
- **Приоритет:** 0.8. Позволяет системе обучаться на накопленном опыте.

## 3. Протокол связи
Доказательный пакет связывается с узлом AST через поле `evidence_link` в формате:
`SOL_EV_LOG://<EVENT_ID>`

При анализе инцидента агент обязан развернуть пакет и предоставить инженеру полный отчет по всем трем слоям.
