# SOL Relations Map

## Общий принцип
SOL является фундаментом (Layer 0), который экспортирует свои аксиомы и правила в прикладные системы, но сам не зависит ни от одной из них.

## Интеграционная матрица

| Система | Роль SOL | Природа взаимодействия |
| :--- | :--- | :--- |
| **E-OS (Runtime)** | Онтологическое ядро | Реализация аксиом SOL в вычислительной среде |
| **Robotics** | Язык описания механики | Описание структурных связей манипуляторов |
| **AI Agents** | Протокол инженерного мышления | Валидация логики действий агентов |
| **CAD/CAE** | Язык геометрической онтологии | Формализация ограничений и потоков |
| **Domain Packs** | Слой знаний (Data) | Использование SOL-примитивов для классификации |

## Правила взаимодействия
1. **Unidirectional Export:** SOL экспортирует стандарты (аксиомы, грамматику). Прикладные системы не имеют права менять правила SOL.
2. **Standardization:** Любая система, использующая SOL, обязана поддерживать целостность графа примитивов (SOURCE, CONSTRAINT, COMPARE, DISSIPATE).
3. **Integration Layer:** Интеграция происходит через формализованные API/Schema, описанные в `core/formal-spec/`.

## Заключение
SOL является «нейтральной зоной» инженерных смыслов. Прикладные системы (такие как E-OS) являются лишь частными реализациями, использующими мощь SOL для решения своих задач.
