# SOURCE

## Definition
Примитив SOL, представляющий собой точку инициации потенциала (энергии, материи или информации) в инженерной системе. SOURCE является «нулевой точкой» причинно-следственной цепи.

## Responsibility
- Инициация процесса внутри системы.
- Предоставление потенциала, необходимого для выполнения работы.
- Поддержание граничных условий входа.

## Inputs / Outputs
- **Input:** Внешний или внутренний поток потенциала (физический или логический).
- **Output:** Инициализированный поток для последующей обработки (через CON::CONNECTION к другим узлам).

## SOL-Canonical Rules
1. **Uniqueness:** Любой замкнутый контур системы обязан иметь как минимум один SOURCE.
2. **Persistence:** SOURCE должен обладать характеристикой стабильности потенциала (если не задано иное).
3. **Dependency:** Ни один объект в системе не может функционировать без связи с SOURCE (прямой или косвенной).

## Anti-Pattern
- **Ghost Potential:** Использование данных или энергии в расчетах без явного определения SOURCE в графе.
- **Infinite Loop Source:** Циклическая зависимость, где SOURCE является своим же производным (бесконечный потенциал без учета энтропии).
- **Domain Pollution:** Включение в определение SOURCE характеристик конкретной отрасли (например, "давление насоса" вместо "поток энергии").
